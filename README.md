# 버핏 봇 — 장기 가치투자 종목추천 / 투자조언 프로그램 (국장 + 미장)

코인 단타 봇과 정반대 결의 봇. **워런 버핏 / 벤저민 그레이엄의 투자 노하우를 정량화**해서
국장(KOSPI)·미장(US)을 같은 잣대로 채점하고, **어느 산업이 괜찮은지**,
**지금 사도 되는 금액대인지**, 나아가 **지금 무엇을·얼마에·왜 사고 무엇을 피할지**까지
조언한다. LLM 호출 없이 100% 로컬 룰, API 키 불필요.

🌐 **웹 앱: https://buffett-bot.streamlit.app/** — 버튼 누를 필요 없이 국장/미장 분석 결과가 항상 떠 있다.

## 핵심 철학
> "훌륭한 기업을 적당한 가격에 사라. 그저 그런 기업을 헐값에 사는 것보다 낫다."
> "남이 탐욕스러울 때 두려워하고, 두려워할 때 욕심내라. 가장 좋아하는 보유기간은 영원히."

품질(75점)과 가격(25점)을 **분리**한다. 좋은 회사라도 비싸면 `우량주·가격 비쌈(대기)`로
빼고 **목표 매수가**를 제시한다. 적자 회사는 자동으로 매수 대상에서 제외한다.

---

## 버핏 체크리스트 — 점수 (총 100점)
| 기둥 | 배점 | 핵심 지표 |
|---|---|---|
| ① 수익성·해자 | 30 | **ROE**, **ROIC**(투하자본이익률, 해자의 진짜 척도), ROE 일관성, 순이익률, 매출총이익률 |
| ② 재무 안정성 | 25 | 부채비율, 유동비율, FCF 수익률 |
| ③ 성장성 | 20 | 매출·순이익 CAGR |
| ④ 밸류에이션·안전마진 | 25 | PER(경기민감주는 **정상화PER**), PBR, 그레이엄수, 배당 |

추가로 **피오트로스키 F-Score(9점)**와 **레드플래그**가 등급·확신도를 보정한다.

등급: `★강력 매수 후보` / `매수 검토` / `우량주·가격 비쌈(대기)` / `관망` / `회피`
확신도: 품질·안전마진·F-Score·위험을 종합한 ★1~5.

## 버핏 정통 지표 (v2에서 추가)
- **ROIC** — 부채 레버리지를 걷어낸 자본효율. 진짜 해자의 척도.
- **오너이익(Owner Earnings)** — 순이익 + 감가상각 − 유지보수 capex. 버핏이 보는 '진짜 현금'.
- **정상화 이익** — 경기민감주(반도체 등)의 사이클 저점/고점 이익 착시를 다년 평균으로 보정
  (삼성전자 PER 50 → 정상화PER로 재평가). 그레이엄·실러(CAPE)의 평균이익 발상.
- **F-Score** — 재무가 좋아지고 있는가(수익성·레버리지·효율 9개 항목).
- **레드플래그** — 적자, 이익 급감, 이익의 질(현금흐름 괴리), FCF 적자, 부채 급증, 주식 희석.
- **금융주 특례** — 은행·보험은 부채·FCF가 사업모델의 일부 → 일반 잣대 끄고 ROE·ROA로 평가.

## "어느 금액대가 좋을지" — 입체 밸류에이션
- **적정가치** = 그레이엄수 · 이익기반(EPS×15) · 오너이익 DCF의 **중앙값**. 안전마진 25% 적용 → 매수권장가.
- **시나리오 DCF** — 약세 / 기본 / 강세 내재가치 **범위**.
- **역DCF** — 현재가는 시장이 향후 몇 % 성장을 가정한 가격인가. 실제 성장보다 낮으면 저평가 신호.
- **기대 연수익률** — 오너이익수익률 + 성장 (멀티플 불변 가정, 버핏·멍거식).

## "어느 산업이 괜찮은지" — 섹터 요약
유니버스를 섹터별로 묶어 평균 품질을 막대그래프로 보여주고, 버핏의 섹터 관점을 주석으로 단다.

## 💼 포트폴리오 조언 (v2 핵심)
- **공포·탐욕 지수(0~100)** — 미장은 CNN Fear & Greed Index 실시간, 국장은 저평가 비율·기대수익률로 자체 산출.
- **버킷 분류** — `✅지금 매수` / `⏳목표가 대기(워치)` / `🚫회피`. 각 종목의 목표가·기대수익률·이유.
- **섹터 분산 제안** — 매수 후보가 한 섹터에 쏠리면 경고.
- **종목 비교(⚖️)** — 2~4개 종목을 나란히 두고 지표별 우승자(⭐)와 4기둥 점수를 한눈에.
- **워치리스트 트리거** — `--watch`로 '대기' 종목의 목표가를 저장, `--check`로 도달 여부 점검.

## 구성
| 파일 | 역할 |
|---|---|
| `buffett.py` | 점수화 엔진(4기둥) + 종합 평가(evaluate) + Verdict |
| `metrics.py` | ROIC·오너이익·정상화이익·F-Score·레드플래그 |
| `valuation.py` | 시나리오 DCF·역DCF·기대수익률·안전마진 |
| `advisor.py` | 종목별 조언 내러티브 + 포트폴리오 차원 제안 |
| `datafetch.py` | yfinance 통합 수집(국장·미장) + 다년 시계열 + 당일 캐시 |
| `universe.py` | 섹터별 우량주 유니버스 |
| `watchlist.py` | 목표 매수가 트리거 |
| `screen.py` | 실행 CLI |
| `app.py` | **웹 UI(Streamlit)** — 사전수집 결과 즉시 표시, 랭킹·섹터·포트폴리오·상세 |
| `precompute.py` | 전체 유니버스를 점수·적정가까지 계산해 결과 JSON 저장(자동 수집용) |
| `results_io.py` | 결과 직렬화/로드 + **데이터 보호 안전장치**(빈 수집이 좋은 데이터 덮어쓰기 방지) |
| `keep_alive.py` | Streamlit 절전 방지(헤드리스 브라우저로 깨우기) |
| `_smoketest.py` | CI 스모크 테스트(문법·모듈·데이터·전 경로 렌더) |

## 🏭 웹 앱 운영 (프로덕션)
- **즉시 로드**: GitHub Actions가 하루 2회(오전 6시·오후 6시 KST) 전체 종목을 사전 분석해
  `cache/results_{kr,us}.json`에 저장 → 앱은 이 파일을 읽어 **버튼 없이 즉시** 표시.
- **데이터 보호**: 수집이 0개거나 직전의 60% 미만이면(야후 rate-limit 사고) **덮어쓰기를 거부**하고
  마지막 정상 데이터를 유지. 30시간 넘게 갱신이 없으면 앱이 신선도 경고를 띄운다.
- **자동 푸시 견고화**: 결과 커밋 시 충돌하면 rebase 후 최대 5회 재시도, 동시성 잠금, 40분 타임아웃.
- **CI 게이트**: `main` push마다 `_smoketest.py`가 전 경로를 헤드리스로 렌더해 예외 0건을 확인(`.github/workflows/ci.yml`).
- **절전 방지**: 6시간마다 `keep_alive.py`가 앱을 깨운다.
- **시크릿**: DART API 키는 코드에 하드코딩 금지 — 환경변수/Streamlit·GitHub 시크릿으로만 주입.

## 실행
```bash
pip3 install yfinance pandas numpy

python3 screen.py --market kr            # 국장 랭킹 + 섹터요약 + 포트폴리오 조언 + 상세
python3 screen.py --market us            # 미장
python3 screen.py --market all           # 통합
python3 screen.py 005930.KS AAPL KO      # 특정 종목 상세 조언 (국장은 .KS/.KQ)
python3 screen.py --market us --top 12   # 상세 조언 상위 12개
python3 screen.py --market kr --watch    # '대기' 종목을 워치리스트에 등록
python3 screen.py --check                # 워치리스트 목표가 도달 점검
python3 screen.py --market kr --no-cache # 당일 캐시 무시하고 새로 수집
```

## 데이터
- **유일 소스: yfinance** — 국장(.KS/.KQ)·미장 모두 커버, 키 불필요.
- 국장은 PER/PBR/EPS가 비어 오는 경우가 많아 재무제표(손익+재무상태표+현금흐름)에서 직접 계산.
- 야후 429(rate limit) 대비 재시도 + **당일 로컬 캐시**(`cache/`). 캐시 버전(`CACHE_VERSION`)으로 스키마 갱신.

## 알아둘 한계 (정직하게)
- **데이터 깊이**: yfinance 무료 데이터는 보통 최근 4~5개 회계연도. 버핏의 10년치는 아니다.
- **데이터 품질**: 일부 종목은 야후의 주식수/가격 단위 오류로 적정가가 비현실적으로 나올 수 있다
  (과도 고평가 시 봇이 '관심 보류'로 안내). 이상하면 원문(국장 DART, 미장 SEC)을 확인할 것.
- **DCF는 가정에 민감**: 성장·할인율 가정이 결과를 좌우한다. 시나리오 범위와 확신도(★)를 함께 보라.
- 정성 요소(경영진의 정직성·자본배분, 해자의 지속성, 산업 구조 변화)는 숫자로 못 잡는다.
